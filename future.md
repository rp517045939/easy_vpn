# easy_vpn 后续打磨路线

这份文档用于记录 easy_vpn 后续的长期演进方向。目标不是一次性做完，而是按风险、收益和使用频率慢慢推进：先让它稳定可靠，再补齐安全边界，然后提升管理体验，最后扩展成更完整的个人远程访问基础设施。

## 一、阶段目标

### Phase 1：稳定可用

目标：让当前 HTTP、TCP、RDP 隧道在日常使用中足够可靠，遇到断线、重启、网络波动时能自动恢复。

- 补齐服务端和客户端的健康检查接口。
- 增加客户端在线、离线、重连、规则同步失败的明确状态。
- 为 WebSocket 连接增加更清晰的断线原因日志。
- 给 TCP/RDP 长连接增加空闲超时和异常关闭清理。
- 对 HTTP 隧道增加请求大小限制和超时配置。
- 为服务端规则文件增加自动备份，避免误操作或写入中断造成规则丢失。
- 增加最小化的端到端测试：HTTP 代理、TCP 转发、客户端重连、规则下发。

验收标准：

- NAS 管理页面、SSH、RDP 可以连续使用一周，服务端和客户端无需人工重启。
- 客户端断网后恢复网络，能在 1 分钟内自动恢复隧道。
- 服务端重启后，在线客户端能自动重连并重新拉取规则。

### Phase 2：安全加固

目标：把远程入口从“能用”推进到“敢长期暴露在公网”。

- 管理面板增加登录失败限流和临时锁定。
- 支持两步验证，优先考虑 TOTP。
- Client Token 支持轮换、过期时间和按设备独立 token。
- 管理 API 增加操作审计日志，包括登录、规则新增、规则修改、规则删除、启停规则。
- 管理面板和 API 增加 CSRF、CORS、Cookie 安全属性检查。
- 隧道规则增加访问控制：来源 IP 白名单、可选 Basic Auth、可选 Header 认证。
- TCP 规则增加允许来源 IP 段配置。
- 敏感配置检查：启动时检测默认密码、弱 token、过宽端口范围并给出警告。
- 日志脱敏：避免 token、密码、Authorization header 出现在日志里。

验收标准：

- 管理面板无法被简单暴力破解。
- 每台客户端可以单独吊销，不影响其他设备。
- 能追踪“谁在什么时候改了哪条规则”。
- 公开暴露的 HTTP 服务可以按规则加一层访问保护。

### Phase 3：可观测性和运维体验

目标：出现问题时能快速定位，而不是靠猜。

- 管理面板展示服务端版本、启动时间、在线设备数、活跃连接数。
- 展示每条规则的近期请求量、流量、错误数、最后访问时间。
- 增加服务端和客户端日志查看页面，支持按级别、设备、规则过滤。
- 增加告警机制：客户端离线、规则连续失败、磁盘空间不足、证书即将过期。
- 支持通过配置接入邮件、Webhook 或其他通知渠道。
- 增加一键导出诊断包：配置摘要、版本、最近日志、规则摘要、系统信息。
- 增加数据保留策略：流量统计按日聚合，旧明细自动清理。

验收标准：

- 某个服务访问失败时，能在面板上看出是客户端离线、本地端口不通、规则不存在，还是服务端转发异常。
- 关键设备离线后能主动通知。
- 日常排障不需要频繁登录服务器执行命令。

### Phase 4：管理面板体验

目标：让新增设备、新增服务、排查问题都更顺手。

- 增加设备详情页：客户端版本、系统类型、运行时长、规则列表、流量统计。
- 增加规则向导：HTTP 服务、SSH 服务、RDP 服务按场景填写。
- 增加端口占用检查和推荐端口选择。
- HTTP 规则支持自定义域名、子域名预览和证书状态提示。
- 规则修改前展示影响范围，删除前二次确认。
- 增加规则搜索、标签、备注和排序。
- 增加暗色模式和移动端适配。
- 客户端本地 Web UI 增加配置校验、连接测试、日志下载。

验收标准：

- 新增一个 NAS Web 服务不需要看文档也能完成。
- 面板可以清楚区分在线设备、离线设备、启用规则、停用规则、异常规则。
- 手机上可以完成查看状态和启停规则等轻量操作。

### Phase 5：部署和升级

目标：降低维护成本，减少手工操作造成的风险。

- 服务端部署脚本增加 dry-run 模式，只检查不修改。
- 部署前自动备份 `.env`、Nginx 配置、规则文件。
- 增加版本号和迁移机制，支持规则结构和数据库结构平滑升级。
- 增加一键回滚最近一次部署的能力。
- 客户端增加自动更新或半自动更新机制。
- 提供 systemd、launchd、Windows Task Scheduler 的完整模板。
- Docker 镜像增加多架构构建，覆盖 x86_64 和 arm64。
- 增加生产环境部署检查清单。

验收标准：

- 更新服务端前可以自动备份，更新失败可以快速恢复。
- 新设备安装客户端只需要复制配置、启动服务、在面板确认上线。
- NAS、Mac、Windows 的客户端守护方式都有清晰路径。

### Phase 6：性能优化

目标：让高频访问、大文件传输、远程桌面等场景更顺滑。

- 对 WebSocket 消息编码进行压测，评估 JSON + base64 的开销。
- 为大响应和大请求增加流式转发，减少内存峰值。
- TCP 转发增加背压控制，避免慢连接拖垮内存。
- RDP 场景继续优化 UDP 通道的清理、超时和统计。
- 增加连接池或复用策略，减少 HTTP 本地转发的连接开销。
- 支持按规则限速，避免单个服务占满上行带宽。
- 增加压测脚本：HTTP 并发、TCP 长连接、RDP 长时间使用、客户端频繁重连。

验收标准：

- 大文件下载不会造成服务端或客户端内存明显飙升。
- 多个 HTTP 请求并发访问时，管理面板和隧道不互相影响。
- RDP 长时间使用后没有大量残留 channel。

### Phase 7：高级能力

目标：把它从“穿透工具”逐步扩展成个人远程访问平台。

- 支持多用户和角色权限：管理员、只读用户、设备维护用户。
- 支持按用户授权访问指定规则。
- 支持浏览器内 SSH 终端。
- 支持浏览器内文件管理，优先从 SFTP/WebDAV 网关开始。
- 支持服务收藏夹和快速入口页。
- 支持设备分组：家庭、云服务器、客户设备、临时设备。
- 支持临时分享链接，带过期时间和访问次数限制。
- 支持多服务端节点，为不同地区或不同网络提供入口。
- 支持与其他隧道方案混合使用，按服务选择最合适的访问路径。

验收标准：

- 常用服务可以从一个入口页快速打开。
- 可以给临时访问者提供有限时、有限权的访问入口。
- 多设备、多服务规模扩大后，管理仍然清晰。

## 二、推荐优先级

### 近期优先做

1. 管理面板登录限流。
2. 独立设备 token 和 token 吊销。
3. 规则文件自动备份。
4. 客户端离线告警。
5. HTTP/TCP/RDP 端到端测试。
6. 规则级访问控制。
7. 服务端部署前备份和 dry-run。

### 暂时不急

1. 多用户权限系统。
2. 浏览器内 SSH 和文件管理。
3. 多节点调度。
4. 临时分享链接。
5. 自动更新系统。

这些能力很有价值，但会明显扩大系统复杂度。建议在基础稳定性和安全边界做扎实之后再推进。

## 三、安全基线清单

长期暴露公网前，至少完成以下检查：

- 管理员密码不是默认值，也不是常见弱密码。
- `SECRET_KEY` 和 `CLIENT_TOKEN` 足够随机，长度不少于 32 位。
- 管理面板启用 HTTPS。
- 不直接暴露服务端内部 HTTP 端口。
- 云服务器防火墙只开放必要端口。
- TCP 端口范围不要过宽，按需开放。
- 客户端 token 泄露后可以单独吊销。
- 关键操作有审计日志。
- 日志不记录密码、token、Authorization header。
- 规则变更有备份，误删可以恢复。

## 四、开发习惯

- 每次新增协议能力，都补一个最小端到端测试。
- 每次新增公网入口，都同步考虑认证、限流、日志和关闭方式。
- 每次新增配置项，都给出默认值、示例和启动时校验。
- 每次修改规则结构，都提供迁移脚本或兼容读取逻辑。
- 面板功能优先服务真实使用场景，避免过早做复杂后台系统。
- 对长期运行任务保持克制：明确超时、退出条件和资源清理。

## 五、里程碑建议

### Milestone A：自己安心用

- 稳定运行一周。
- 客户端自动重连可靠。
- 规则文件自动备份。
- 管理面板有登录限流。
- 基础端到端测试通过。

### Milestone B：公网长期运行

- 支持独立设备 token。
- 支持 token 吊销。
- 支持规则级访问控制。
- 有操作审计日志。
- 有离线告警和诊断信息。

### Milestone C：家庭基础设施

- 面板能完成大部分日常运维。
- 支持设备详情、规则统计、日志查看。
- 支持部署备份和快速回滚。
- 支持多平台客户端守护运行。

### Milestone D：个人远程访问平台

- 支持多用户和权限。
- 支持浏览器内 SSH 或文件管理。
- 支持临时授权访问。
- 支持多入口或混合访问路径。

## 六、当前建议

下一步最值得做的是安全和可恢复性：

1. 先做管理面板登录限流，降低公网入口风险。
2. 再做规则文件自动备份，避免配置误删或损坏。
3. 然后做独立设备 token，为后续设备管理打基础。
4. 最后补端到端测试，让后续改动更敢下手。

这几项完成后，easy_vpn 就会从“能跑起来的工具”向“可以长期依赖的个人基础设施”迈出比较扎实的一步。
